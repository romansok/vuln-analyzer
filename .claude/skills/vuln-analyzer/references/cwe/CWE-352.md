# CWE-352: Cross-Site Request Forgery (CSRF)

## Plain-English explanation
A logged-in user visits an attacker-controlled page. The page silently submits a request to your site — using the user's cookies. Your server can't tell the difference between this and a legitimate click, so it executes the state-changing action: change email, transfer money, add an admin.

## Typical attack pattern
A banking site has a `POST /transfer` endpoint that accepts a form-encoded body with `to_account` and `amount`. The attacker emails the victim a link to `evil.com`. The page on `evil.com` auto-submits a hidden form to `https://bank.com/transfer`. The browser attaches the user's session cookie. Money moves.

## Reachability hints
- Cookie-based session auth (the most common shape that's vulnerable). Bearer-token APIs hit from `fetch` with `Authorization` headers are generally **not** vulnerable.
- Forms without a CSRF token (`csurf`, `flask-wtf`, framework defaults disabled).
- Cookies set without `SameSite=Lax` or `SameSite=Strict`.
- State-changing GETs (`GET /delete?id=…`) — always wrong shape; doubly wrong here.

## Common workarounds when a bump is blocked
- Set `SameSite=Lax` (default in modern browsers) or `Strict` on the session cookie. That alone blocks most cross-site form posts.
- Add an anti-CSRF token: random per-session, embedded in the form, validated on submit.
- Require a custom header (`X-Requested-With: XMLHttpRequest`) on state-changing endpoints — browsers won't send it cross-origin without CORS preflight.
- Use POST/PUT/DELETE for state changes; never GET.

## Concrete code example

```js
// Vulnerable — cookie auth, no CSRF protection
app.post('/transfer', (req, res) => {
  transfer(req.session.userId, req.body.to, req.body.amount);
  res.json({ ok: true });
});

// Safer — SameSite + token
app.use(session({ cookie: { sameSite: 'lax', secure: true, httpOnly: true } }));
app.use(csurf());
app.post('/transfer', (req, res) => {
  // csurf middleware rejects requests without a valid token
  transfer(req.session.userId, req.body.to, req.body.amount);
  res.json({ ok: true });
});
```
