# CWE-770: Allocation of Resources Without Limits (incl. ReDoS)

## Plain-English explanation
There's no upper bound on memory, CPU, or file handles consumed per request. A pathological input (long regex backtrack, deep recursion, huge array allocation) tips the worker into either OOM or running for so long that the request pool exhausts. Close cousin to CWE-400; this is the structural / library shape.

## Typical attack pattern
A login form runs the email through a regex like `/^([a-z0-9]+)+@example\.com$/`. The attacker submits a 30-character string of `a`s with one non-matching tail character. The regex engine backtracks through 2^30 possibilities. The worker pegs a CPU for ~30 seconds. A few of these tank the service.

## Reachability hints
- `String.matches`, `RegExp.test`, `re.match` on user input — especially with patterns containing `(a+)+`, `(a|a)+`, `(.*)*`.
- Linters/parsers (`marked`, `acorn`, …) without size limits.
- `JSON.parse` / `xml` / `yaml` with no depth limit.
- Server-side image resize with no input-dimension cap.
- Library-side: many ReDoS CVEs come through 3rd-party regex helpers in date-parsers, color-parsers, URL-parsers.

## Common workarounds when a bump is blocked
- Switch the regex engine to one with linear guarantees: `re2`, `re2-wasm`, Hyperscan. The same pattern that times out in PCRE runs in microseconds in re2.
- Cap input size **before** the regex runs: `if (s.length > 256) reject`.
- For parsers, set hard limits: max depth, max entries, max string length.
- Add a per-request timeout (Node `setTimeout` + `AbortController`, Python signal alarm) so one pathological input can't burn a worker indefinitely.

## Concrete code example

```js
// Vulnerable — catastrophic backtracking
const EMAIL = /^([a-z0-9]+)+@example\.com$/;
app.post('/login', (req, res) => {
  if (!EMAIL.test(req.body.email)) return res.status(400).end();
  ...
});

// Safer — bounded input + linear engine
const RE2 = require('re2');
const EMAIL = new RE2('^[a-z0-9]+@example\\.com$');
app.post('/login', (req, res) => {
  if (typeof req.body.email !== 'string' || req.body.email.length > 64) return res.status(400).end();
  if (!EMAIL.test(req.body.email)) return res.status(400).end();
  ...
});
```
