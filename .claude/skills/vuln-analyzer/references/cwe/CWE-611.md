# CWE-611: XML External Entity (XXE)

## Plain-English explanation
The app parses XML from an untrusted source with a parser that resolves "external entities". The attacker embeds entity references that pull in remote URLs, read local files, or expand exponentially. The parser obediently fetches/expands them during parsing.

## Typical attack pattern
A SOAP endpoint accepts XML payloads. The attacker submits:
```xml
<!DOCTYPE foo [ <!ENTITY xxe SYSTEM "file:///etc/passwd"> ]>
<request><name>&xxe;</name></request>
```
The XML parser resolves `&xxe;` by reading `/etc/passwd` from the server's filesystem and includes the contents in the parsed `name` field. The handler returns "Hello, &xxe;" — and the attacker reads the passwd file.

## Reachability hints
- Any XML parser with defaults: Java `DocumentBuilderFactory`, Python `xml.etree.ElementTree` (older), `lxml.etree.XMLParser`, libxml2 callers, `DOMParser` from XML libs.
- SOAP/SAML/XML-RPC endpoints (the format itself).
- File-upload handlers that read SVGs / OOXML / EPUB — they contain XML inside.
- Old SOAP libraries are notorious; many CVEs cluster here.

## Common workarounds when a bump is blocked
- Disable external entity resolution at parser construction. Examples:
  - Java: `factory.setFeature("http://apache.org/xml/features/disallow-doctype-decl", true);`
  - Python `lxml`: `XMLParser(resolve_entities=False, no_network=True)`
  - Python stdlib: switch to `defusedxml`.
- Disable DTD processing entirely if the app doesn't need it.
- For SVG uploads: re-encode through a renderer that produces clean SVG (or convert to PNG/JPEG server-side).
- Network-egress restrictions on the worker — if the server can't reach `file://` or random hosts, exfiltration is harder.

## Concrete code example

```python
# Vulnerable
import xml.etree.ElementTree as ET
tree = ET.fromstring(body)           # older versions process entities

# Safe
from defusedxml.ElementTree import fromstring
tree = fromstring(body)              # rejects DTDs/entities by default
```
